20260422🏆 Etrip 專案更新摘要
1. 住宿資訊升級 (Multi-Hotel Support)
支援複數飯店：後端 AI 解析與前端資料結構已同步升級。現在支援在同一次行程中解析多間不同的入住飯店，並依序顯示在飯店卡片中。

視覺優化：飯店卡片改用列表形式呈現，名稱加粗並放大至 18px，閱讀更直覺。

2. 地理資訊與距離運算 (Geographic Intelligence)
備案池公里數顯示：備案池中的每個地點會自動計算與當天行程「最後一個景點」的直線距離（KM），字體優化為 13px 白色加粗，方便判斷排程合理性。

自動座標補完：無論是手動新增或 AI 抽取的備案景點，系統都會自動呼叫 API 補齊經緯度資訊。

3. 操作自動化 (Automation)
🗺 一鍵生成路線：新增綠色功能按鈕，點擊後自動將整天行程串接成 Google Maps 導航連結，一鍵開啟整日的行車/步行路線。

動態天數按鈕：備案池按鈕不再侷限於 Day1-Day4，會根據行程實際天數自動生成對應數量的 D1~DN 按鈕。

4. UI 體驗與穩定性修復 (UX & Bug Fixes)
摺疊狀態記憶：引入 expandedDays 狀態管理。現在修改行程內容、調整排序後，原本打開的天數卡片會保持展開，不再尷尬地全部縮合。

載入順序優化：修正 index.html 的 Script 載入順序，徹底解決 Sortable 與 renderTrip 未定義的報錯問題。

防呆機制：在 api.js 中增加了資料型態檢查，避免後端回傳格式異常導致網頁當掉。
